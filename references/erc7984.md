# ERC-7984 — Confidential Token Standard

ERC-7984 is the confidential token standard co-developed by Zama and OpenZeppelin.
All balances and transfer amounts are encrypted as `euint64`. It is NOT ERC-20 compatible.

## Required Inheritance

```solidity
// ALWAYS: ZamaEthereumConfig first, ERC7984 second
contract MyToken is ZamaEthereumConfig, ERC7984, Ownable2Step {
    constructor(address owner, string memory name, string memory symbol, string memory uri)
        ERC7984(name, symbol, uri)
        Ownable(owner)
    {}
}
```

## Full Interface

```solidity
// ── Read ──────────────────────────────────────────────────────────────────────
// Returns encrypted handle — NOT a plaintext number
function confidentialBalanceOf(address account) external view returns (euint64);
function confidentialTotalSupply() external view returns (euint64);

// ERC-20 compatible metadata (plaintext)
function name()        external view returns (string memory);
function symbol()      external view returns (string memory);
function decimals()    external view returns (uint8);    // recommended: 6
function contractURI() external view returns (string memory);

// Operators (boolean — not encrypted allowances)
function isOperator(address holder, address spender) external view returns (bool);

// ── Write ─────────────────────────────────────────────────────────────────────
// Transfer with new encrypted input (most common)
function confidentialTransfer(address to, externalEuint64 amount, bytes calldata proof) external returns (bool);

// Transfer using an existing handle (caller must have ACL access to `amount`)
function confidentialTransfer(address to, euint64 amount) external returns (bool);

// Transfer from another account (requires msg.sender to be approved operator)
function confidentialTransferFrom(address from, address to, externalEuint64 amount, bytes calldata proof) external returns (bool);

// Operator approval (binary, not amount-based)
function setOperator(address operator, bool approved) external;
```

> **Overload note in ethers.js:** When two `confidentialTransfer` signatures exist,
> call using the full selector string:
> `token["confidentialTransfer(address,bytes32,bytes)"](to, handle, proof)`

## Minimal Mintable Token

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.27;

import {FHE, externalEuint64} from "@fhevm/solidity/lib/FHE.sol";
import {Ownable2Step, Ownable} from "@openzeppelin/contracts/access/Ownable2Step.sol";
import {ERC7984} from "@openzeppelin/confidential-contracts/token/ERC7984/ERC7984.sol";
import {ZamaEthereumConfig} from "@fhevm/solidity/config/ZamaConfig.sol";

contract ERC7984Token is ZamaEthereumConfig, ERC7984, Ownable2Step {
    constructor(address owner, string memory name, string memory symbol, string memory uri)
        ERC7984(name, symbol, uri) Ownable(owner) {}

    // Encrypted mint — amount never visible to owner
    function mint(address to, externalEuint64 enc, bytes calldata proof) external onlyOwner {
        _mint(to, FHE.fromExternal(enc, proof));
    }

    // Plaintext mint — use only for initial distribution in trusted contexts
    function mintClear(address to, uint64 amount) external onlyOwner {
        _mint(to, FHE.asEuint64(amount));
    }

    function burn(address from, externalEuint64 enc, bytes calldata proof) external onlyOwner {
        _burn(from, FHE.fromExternal(enc, proof));
    }
}
```

> Full implementation with NatDoc: `assets/contracts/ERC7984Token.sol`

## ERC-20 ↔ ERC-7984 Wrapper

Wraps any ERC-20 token into a confidential ERC-7984 token at a configurable rate.

```solidity
import {ERC7984ERC20Wrapper} from "@openzeppelin/confidential-contracts/token/ERC7984/extensions/ERC7984ERC20Wrapper.sol";
import {IERC20} from "@openzeppelin/contracts/interfaces/IERC20.sol";

contract WrappedUSDC is ZamaEthereumConfig, ERC7984ERC20Wrapper {
    constructor(IERC20 usdc)
        ERC7984ERC20Wrapper(usdc, 1)  // 1:1 rate
        ERC7984("Confidential USDC", "cUSDC", "https://example.com/cusdc")
    {}
    // wrap(address to, uint256 amount)  — user calls approve() first
    // unwrap(externalEuint64, bytes)    — async, goes through Gateway
}
```

## Available Extensions

| Extension | Import Path Suffix | What it adds |
|---|---|---|
| `ERC7984ERC20Wrapper` | `/extensions/ERC7984ERC20Wrapper.sol` | Wrap/unwrap ERC-20 ↔ ERC-7984 |
| `ERC7984Freezable` | `/extensions/ERC7984Freezable.sol` | Freeze/unfreeze accounts |
| `ERC7984ObserverAccess` | `/extensions/ERC7984ObserverAccess.sol` | Observer can view transfers |
| `ERC7984Restricted` | `/extensions/ERC7984Restricted.sol` | Transfer restrictions |
| `ERC7984Votes` | `/extensions/ERC7984Votes.sol` | Confidential governance voting power |

All import from `@openzeppelin/confidential-contracts/token/ERC7984/`.

## Key Behaviors

- **No-revert transfers:** Insufficient balance → 0 transferred, no revert, no info leak
- **Uninitialized handle:** `confidentialBalanceOf` returns `bytes32(0)` before first mint
- **Operator model:** Binary (`setOperator(addr, true/false)`), not ERC-20 allowances
- **Decimals:** ERC-7984 recommends `decimals() = 6` (same as USDC)
